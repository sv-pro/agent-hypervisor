Root cause

A configuration change to the rate-limit service was deployed without the usual canary stage, causing the gateway to reject all authenticated requests.

Remediation

  • Require canary stage for all rate-limit config changes
  • Add pre-deployment integration test for rate-limit rules
  • Update runbook with rollback procedure
